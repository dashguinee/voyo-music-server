THE PHYSICS (Why It Feels Real)
═════════════════════════════════════════════════════════════════════════════

Everything here is based on actual neon sign behavior:

1. STARTUP FLICKER - Real tubes don't light instantly. They flicker
   chaotically below ignition voltage. Implemented as non-uniform
   keyframes with irregular intervals.

2. THREE-LAYER BLOOM - Light scatters in 3 patterns: sharp core,
   medium bloom, soft halo. Real optical physics. Layered text-shadow.

3. WARM vs COLD - Red/orange neons are direct gas emission. Blue/
   green/purple are argon + phosphor coating. Different shadow
   behavior reflects actual physics.

4. RAIN/FOG - Water droplets scatter light unpredictably. Bloom
   extends 2-3x. Color desaturates. Chromatic aberration happens.

5. DEGRADATION - Real signs fail: gas leaks (dimming), electrode wear
   (reduced glow), phosphor aging (color shift), air contamination
   (random flicker).

6. AC RIPPLE - 60Hz electrical current creates subtle breathing
   effect. Every powered neon has this.

WHEN THINGS GO WRONG
═════════════════════════════════════════════════════════════════════════════

Glow too dim?
  → Use dark background (#000-#0a0a0a)
  → Add .high-power class
  → Or increase text-shadow opacity

Glow too sharp?
  → Increase shadow spread: 0 0 15px → 0 0 25px
  → Add more shadow layers

Animation stutters?
  → Remove animation, use static glow
  → Reduce number of elements
  → Use .low-power variant

Doesn't look right?
  → Make sure color matches shadow hex
  → Check dark background
  → Try different colors

Full help:
  → See "Troubleshooting" in NEON_QUICK_REFERENCE.md
